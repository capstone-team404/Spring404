# 🗺️ 여기지! (여성 기행 지켜!)

> 여성 1인 여행자를 위한 DB 기반 가이드맵 및 안전 동행 매칭 서비스

여기지! (여성 기행 지켜 !)는 여성 1인 여행자의 불안감을 줄이고 안전한 여행을 지원하기 위해 설계된  
**데이터 기반 안전 여행 지원 웹**입니다.

사용자들이 직접 작성한 안전 리뷰 데이터와 공공 데이터를 기반으로  
여행지의 **안전도를 지도에 시각화**하고, **안전한 여행 경로를 추천**합니다.

또한 동일 지역의 여성 여행자 간 **동행 매칭 기능**을 통해  
혼자 여행하는 상황에서도 심리적·물리적 안전을 확보하는 것을 목표로 합니다.

## Why Hereji?

여성 1인 여행자는 다음과 같은 문제를 겪습니다:

- 여행지에 대한 **신뢰할 수 있는 안전 정보 부족**
- 낯선 지역에서의 **불안감과 심리적 부담**

→ 기존 여행 플랫폼은 "안전" 중심 정보가 부족합니다.

## What We Built

본 프로젝트는 아래 3가지 핵심 기능으로 문제를 해결합니다:

### 🗺️ 안전 데이터 기반 지도 시각화

- 공공 데이터 및 사용자 리뷰 기반 안전 점수 DB 구축
- Google Maps 기반 안전도 시각화

### 🧭 AI 기반 안전 여행 경로 추천

- OpenAI API를 활용한 안전 경로 생성
- 안전도 + 위치 반영

### 👭 여성 전용 동행 매칭

- 같은 지역/시간 여행자 매칭
- 안전 인증 기반 사용자 필터링

## MVP

본 프로젝트의 핵심 MVP는 **데이터 기반 안전 지도 (Safety Map)** 입니다.

- 핵심 기능: 안전 지도 시각화 및 안전 경로 추천 시스템

사용자 리뷰 데이터 입력 (위험/안전 경험)  
공공 데이터(지역 범죄율, 거리 활성도 등) 연동  
두 데이터를 통합하여 **장소별 안전 점수 산출**  
이를 기반으로 지도 위에 **안전도 시각화**  
안전 점수를 기반으로 **안전 경로 추천 기능 제공**

- 결과:

사용자는 특정 지역의 안전도를 **직관적으로 확인 가능**  
“어디가 안전한가?”를 지도에서 판단 가능  
출발지–목적지 간 **더 안전한 이동 경로 선택 가능**

- 핵심 가치:

여성 1인 이동 시 발생하는 **불안감 최소화**  
안전 정보를 기반으로 한 **자율적 이동권 강화**  
단순 길찾기가 아닌 **안전 중심 경로 선택** 가능

## Core Logic

### 🗺️ 안전 지도 생성 로직 (Safety Map Generation)

본 시스템의 핵심 로직은 사용자 리뷰 데이터 + 공공 데이터 분석을 통한 안전도 점수 산출 및 지도 시각화입니다.

1. 데이터 수집  
사용자 리뷰 데이터 (위험/안전 경험, 텍스트 입력)  
공공 데이터 (치안 정보, 사고 발생률, 환경 데이터 등)

2. 데이터 전처리  
리뷰 텍스트 정제 (불용어 제거, 키워드 추출)  
지역 단위로 데이터 정렬 및 통합

3. 안전도 점수 계산  
사용자 리뷰 기반 점수  
공공 데이터 기반 위험도 점수

4. 안전 지도 생성  
Google Maps 기반 지도 렌더링  
지역별 safety_score를 평점으로 변환

## Repository Structure

본 프로젝트는 **Spring404 레포지토리 기반 멀티 브랜치 구조**로 관리되며,  
프론트엔드 / 백엔드 / AI 모듈이 각각 독립 브랜치에서 개발 및 관리됩니다.

### 🌿 GitHub 구조

```text
📦 Spring404 (main repository)
┣ 🌿 frontend
┃ ┣ App.jsx
┣ 🌿 backend
┃ ┣ .gitignore
┃ ┣ db.py
┃ ┣ main.py
┃ ┣ schemas.py
┃ ┗ requirements.txt
┣ 🌿 ai
┃ ┣ .gitignore
┃ ┣ README.md
┃ ┣ main.py
┃ ┗ requirements.txt
┗ 🌿 main
  ┗ 통합 관리 및 배포 브랜치
```

### 📁 각 브랜치 역할

#### 🌿 frontend 브랜치

`App.jsx`
- React 기반 사용자 UI
- 지도 (Google Maps) 렌더링
- 안전 지도 시각화 화면

#### 🌿 backend 브랜치

`.gitignore`
- 깃허브에 업로드하지 않을 파일 목록 관리
- venv, 캐시 파일, 환경변수 등 제외 설정

`db.py`
- MySQL 데이터베이스 연결 및 관리
- 리뷰 데이터 저장 및 조회 기능 구현

`main.py`
- FastAPI 기반 백엔드 서버 실행 파일
- API 엔드포인트 정의 및 요청 처리
- AI 서버와 통신하여 위험도 계산

`schemas.py`
- 요청/응답 데이터 구조 정의 (Pydantic)
- 데이터 검증 및 타입 관리

`requirements.txt`
- 프로젝트 실행에 필요한 라이브러리 목록
- 동일한 개발 환경 재현을 위한 설정 파일

#### 🌿 ai 브랜치

- OpenAI API 연동
- 안전 여행 경로 로직
- 데이터 기반 분석 프롬프트 처리

→ 구조 설계 목적:

- 팀원별 독립 개발 환경 보장
- 기능 단위 분리로 협업 효율 증가
- 이후 main 브랜치에서 통합 관리
